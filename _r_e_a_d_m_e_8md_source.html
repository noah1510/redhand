12 The goal is to create an easy to use 2D Game engine using C++.
13 Its purpose is to give new programmers an easy introduction into game programming and even more abstract topics like object oriented programming and graphics programming in general.
14 The used OpenGL APIs are GLFW and glad so this project should run on all operating systems.
15 **This Project is still in the alpha phase and there might be srastic changes to the codebase and API causing your Programs to break with an update.
16 Backwards compatibility will be a consideration after the release of version 0.1.0 everything prior to that might include drastic changes!**
17 This Project provides several functions and objects that make it easier to start building your own games.
18 While these objects give you a simple way to get started, they also allow you to insert advanced OpenGL commands.
19 
20 You can find all the documentation [here](https://noah1510.github.io/redhand/).

28 ## How to compile/install this project
29 
30 There are two release types tagged and latest.
31 The tagged version is generally more stable and tested while the latest version has all the features but might break some games.
32 
33 ### Ubuntu (bionic) based distributions
34 
35 There are two personal package archieves you can choose from:
36 
37 * [libredhand tagged](https://launchpad.net/~noasakurajin/+archive/ubuntu/libredhand)
38 * [libredhand latest](https://launchpad.net/~noasakurajin/+archive/ubuntu/libredhand-latest)
39 
40 ### Windows
41 
42 For an mostly automated install check the [manual Installation](https://noah1510.github.io/redhand/md_doc_markdown_install.html).
43 
44 You can download the either the latest or the last tagged realese form [github](https://github.com/noah1510/redhand/releases).
45 The dlls are automatically compiled and uploaded on every tagged and versioned commit.
46 You can just include those dlls and the headers to you project to build your program using redhand.
47 You might also need to install glfw to get it working properly
48 **Make sure your C++ compiler supports the `<filesystem>` include (gcc > 9.2.0 or llvm > 9) since redhand needs it**

55 ### Extra Notes for contibutors
56 
57 All changes have to be made on the dev branch and merged to master via a pull request.
58 Open a Terminal in the folder where you cloned the repository to, type `git checkout dev` and you are ready to go.
59 To update the branch from master first type `git fetch` then `git rebase origin/master`.
60 The CI has to succeed in compiling before you are allowd to push your changes to master.

62 ## Getting started
63 
64 If you want to get started using this project, begin by modifying the games.cpp file.
65 The functions that specify the game logic are here.
66 
67 game_init is called in the beginning to construct the main world that the game will enter first.
68 game_loop is called every game tick, here is where all the logic happens.
